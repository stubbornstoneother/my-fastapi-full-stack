# Frontend Design Skill

This skill guides the creation of distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics. Follow these structured steps and checklists to implement real working code with exceptional attention to aesthetic details and creative choices.

## 1. Design Thinking Checklist

Before coding, understand the context and commit to a **BOLD** aesthetic direction:
- [ ] **Define Purpose**: State what problem this interface solves and who uses it.
- [ ] **Choose Tone**: Pick an extreme tone (e.g., brutally minimal, maximalist chaos, retro-futuristic, organic/natural, luxury/refined, playful/toy-like, editorial/magazine, brutalist/raw, art deco/geometric, soft/pastel, industrial/utilitarian).
- [ ] **Verify Constraints**: Identify technical requirements such as framework, performance, and accessibility.
- [ ] **Ensure Differentiation**: Decide on the ONE feature that makes this unforgettable.

**CRITICAL**: Choose a clear conceptual direction and execute it with precision. The key is intentionality, not intensity.

## 2. Implementation Checklist

Execute production-grade and functional code (HTML/CSS/JS, React, Vue, etc.) based on the aesthetic vision. Match implementation complexity to the aesthetic vision (e.g., Maximalist needs elaborate code; Minimalist needs restraint and precision).

### Typography & Color
- [ ] Choose distinctive, characterful fonts (pair a display font with a refined body font).
- [ ] Establish a cohesive palette using CSS variables (dominant colors with sharp accents).
- [ ] **Avoid** generic fonts (Arial, Inter, Roboto, Space Grotesk, system fonts) and clichéd color schemes (e.g., purple gradients on white).

### Motion
- [ ] Add high-impact motion (e.g., staggered reveals with `animation-delay` on page load).
- [ ] Include surprising scroll-triggering and hover states. Prioritize CSS-only solutions for HTML; use Motion library for React when available.

### Spatial Composition & Visual Details
- [ ] Implement unexpected layouts (e.g., asymmetry, overlap, diagonal flow, grid-breaking elements).
- [ ] Utilize generous negative space OR controlled density.
- [ ] Create atmosphere and depth (gradient meshes, noise textures, geometric patterns, layered transparencies, dramatic shadows, custom cursors, grain overlays).
- [ ] **Never** default to solid colors or predictable, cookie-cutter component patterns.

## 3. Verification Checklist

- [ ] Does the UI feel genuinely designed for the context without converging on common choices?
- [ ] Are light/dark themes, fonts, and aesthetics varied?
- [ ] Does the codebase demonstrate extraordinary creative work and polish?
